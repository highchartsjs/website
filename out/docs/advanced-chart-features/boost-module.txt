module</h1>\n<p>The Boost module is a stripped down renderer and a set of data handling modifications for Highcharts. It bypasses some of the standard Highcharts features, applies some trade-offs and focuses on handling and rendering as many data points as possible as quickly as possible. It renders the data using WebGL on a canvas inside the chart's SVG structure.</p>\n<p>This document will guide you through your first steps with the Highcharts boost module.</p>\n<h2>Including boost in your project</h2>\n<p>The boost module is a \"renderer-in-a-module\". Including <a href=\"https://code.highcharts.com/modules/boost.js\" target=\"_blank\">modules/boost.js</a> will, by default, activate boost for all series that support it (line series, column, bar, treemap, heatmap, scatter, bubble, area).</p>\n<p>If needed, boost can be disabled on a chart-by-chart basis by setting <a href=\"https://api.highcharts.com/highcharts/boost.enabled\" target=\"_blank\">boost.enabled</a> to <code>false</code> in the chart configuration.</p>\n<p><strong>Note</strong>: Boost should be the last module included. This is because it overrides both standard Highcharts functionality, and functionality in certain modules (namely treemap, heatmap, bubble, and scatter).</p>\n<h2>Configuration Options</h2>\n<p>The main boost configuration is set in the <code>boost</code> property in the chart options. See <a href=\"https://api.highcharts.com/highcharts/boost\" target=\"_blank\">the API</a> for the detailed description of available options.</p>\n<pre><code>{\n    boost: {\n        useGPUTranslations: true,\n        // Chart-level boost when there are more than 5 series in the chart\n        seriesThreshold: 5\n    },\n\n    title: {\n        text: 'Highcharts Boost'\n    },\n\n    series: [{\n        boostThreshold: 1,  // Boost when there are more than 1\n                            // point in the series.\n        data: [ [0, 1], [1, 2], [2, 3] ]\n    }]\n};\n</code></pre>\n<p><em>Configuration for a boosted line chart.</em></p>\n<h2>Configuration Data Options</h2>\n<p>In boost mode, <a href=\"https://api.highcharts.com/highcharts/plotOptions.series.turboThreshold\" target=\"_blank\">turbo mode</a> is always turned on. That means all data points should be configured as an array of numbers (e.g. <code>[1, 2, 3]</code>) or a two dimensional array of numbers (e.g. <code>[ [1, 2], [2, 3], [3, 4] ]</code>).</p>\n<p>Note that when <code>dataGrouping</code> is enabled (default in <code>stockChart</code>), boost mode will not kick in.</p>\n<h2>Series boosting versus chart boosting</h2>\n<p>There are two different ways of boosting charts - on a series-by-series level (<a href=\"https://api.highcharts.com/highcharts/plotOptions.series.boostThreshold\" target=\"_blank\">series.boostThreshold</a>), and on the chart as a whole (<a href=\"https://api.highcharts.com/highcharts/boost.seriesThreshold\" target=\"_blank\">boost.seriesThreshold</a>).</p>\n<p>The former works well in most cases, whereas the latter is meant for charts with large amounts of series (such as monitoring server clusters).</p>\n<p>When boosting on the chart level, all boostable series are rendered to the same canvas element, whereas on the series boosting level, each series have their own, final, render target. As such, combination charts that combine both boostable and non-boostable series types should always be boosted on the series-level so that the draw order is as expected.</p>\n<p>The two different modes each has their own threshold value. For chart-level boosting, the threshold is the number of series that must be present in the chart for the boost to “kick in”. For series-level boosting, the threshold is the number of points in the series that must be present for that particular series to enter boost mode.</p>\n<h2>Caveats</h2>\n<p>The boost module contains a WebGL renderer that replaces parts of the SVG renderer. Additionally, it bypasses and simplifies some of the resource-hungry aspects of handling big data. As such, certain features are not available for boosted charts. Most of these features deal with interactivity, such as animation support. But there are a few that relate to visuals as well.</p>\n<ul>\n<li>The largest caveat is that rectangles for column and bar charts are always drawn as a single 1 pixel wide line. This will likely not be the desired outcome when zoomed in to the level where each column/bar is visible as an individual entity. Thus, column and bar charts are more suited to series-level boosting.</li>\n<li>The area of area and areaspline series are drawn as 1px columns. This works well with the intended way of using the boost module, which is that it kicks in when the number of data points crosses the <code>boostThreshold</code>. But if the boost threshold is set too low, an area and areaspline charts will look like a column chart. This is a limitation that we are <a href=\"https://github.com/highcharts/highcharts/issues/14207\" target=\"_blank\">considering fixing</a>. In addition to this, the <em>line</em> itself is not rendered in area and areaspline series.</li>\n<li>Marker shapes, apart from circles, are not supported</li>\n<li>Dash style for lines is not supported</li>\n<li>Stacking, and negative colors are not supported.</li>\n<li>Line width is limited to 1px.</li>\n<li>Point click handlers are not supported in boost mode.</li>\n</ul>\n<p>The intended way of using the module, is to set thresholds in such a way that the SVG-renderer “takes over” rendering when zooming in. This approach gives the expected interactivity when the points are less dense, coupled with high performance when the point density is high.</p>\n<h2>Optimizing tips</h2>\n<ul>\n<li>Set the extremes (<a href=\"https://api.highcharts.com/highcharts/xAxis.min\" target=\"_blank\">min</a> and <a href=\"https://api.highcharts.com/highcharts/xAxis.max\" target=\"_blank\">max</a>) explicitly on the <code>xAxis</code> and <code>yAxis</code> in order for Highcharts to avoid computing the extremes.
