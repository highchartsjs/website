system</a>. Here is a quick overview of the axis elements:</p>\n<p><img src=\"axis_description.png\" alt=\"axis_description.png\"></p>\n<h2>Axis labels, tickmarks and gridlines</h2>\n<p>The axis labels, tickmarks and gridlines are closely linked and all scale together. Their positioning is calculated to best fit the data present in a chart. </p>\n<h3>Ticks</h3>\n<p>Tick marks are the lines placed along an axis to show the units of measurement. The spacing between ticks are mainly decided by the <a href=\"https://api.highcharts.com/highcharts/xAxis.tickInterval\" target=\"_blank\">tickInterval</a> and <a href=\"https://api.highcharts.com/highcharts/xAxis.tickPixelInterval\" target=\"_blank\">tickPixelInterval</a> options. Labels and grid lines are laid out on the same positions as the tick marks. </p>\n<p>The <code>tickInterval</code> option decides the interval of the tick marks in axis units. The tick interval defaults to null, which means it is computed to approximately follow the <code>tickPixelInterval</code> on linear and datetime axes.</p>\n<p>On <strong>categorized</strong> axes, a null <code>tickInterval</code> will default to 1, one category.</p>\n<p>Note that <strong>datetime</strong> axes are based on milliseconds, so for example an interval of one day is expressed as 24 * 3600 * 1000. </p>\n<p>On <strong>logarithmic</strong> axes, the <code>tickInterval</code> is based on powers, so a <code>tickInterval</code> of 1 means one tick on each of 0.1, 1, 10, 100 etc. A <code>tickInterval</code> of 2 means a tick of 0.1, 10, 1000 etc. A <code>tickInterval</code> of 0.2 puts a tick on 0.1, 0.2, 0.4, 0.6, 0.8, 1, 2, 4, 6, 8, 10, 20, 40 etc.</p>\n<p>The <code>tickPixelInterval</code> option sets an approximate pixel interval of the tick marks based on a pixel value (if <code>tickInterval</code> is null). This makes it work well with responsive layouts, ensuring a reasonable distance between ticks regardless of the chart size and axis length. It doesn't apply to categorized axis. Defaults to 72 for the y-axis and 100 for the x-axis.</p>\n<h3>Minor ticks</h3>\n<p>If the <a href=\"https://api.highcharts.com/highcharts/xAxis.minorTickInterval\" target=\"_blank\">minorTickInterval</a> option is set, minor ticks are laid out between the major ones. This includes minor tick marks, and minor grid lines, which have their own options for look and feel, but excludes labels. </p>\n<h3>Labels</h3>\n<p>The axis labels can be found along the axis showing the value of the data it corresponds to. Labels can also be customized using a formatter function:</p>\n<pre><code>yAxis: {\n    labels: {\n        formatter: function() {\n            return this.value + ' %';\n        }\n    },\n},\n</code></pre>\n<p>The above example takes the value of the y-axis label and adds a % symbol at the end of it.</p>\n<h3>Grid lines</h3>\n<p>Grid lines are collections of horizontal (and/or vertical) lines that divide a chart into a grid, making it easier to read values of the chart. </p>\n<p>To enable or disable gridlines for either the x or y-axis, set the <a href=\"https://api.highcharts.com/highcharts/xAxis.gridLineWidth\" target=\"_blank\">gridLineWidth</a> of the respective axis:</p>\n<pre><code>xAxis: {\n    gridLineWidth: 1\n},\nyAxis: {\n    gridLineWidth: 1\n}\n</code></pre>\n<p>Grid lines for the y-axis are enabled by default (<code>gridLineWidth: 1</code>), and disabled by default for the x-axis  (<code>gridLineWidth: 0</code>).</p>\n<p>Other options for grid lines can be found in the <a href=\"https://api.highcharts.com/highcharts/xAxis\" target=\"_blank\">API reference</a> for the x and y-axis.</p>\n<p>Minor grid lines are intermediary lines that can be enabled by setting the <a href=\"https://api.highcharts.com/highcharts/xAxis.minorTickInterval\" target=\"_blank\">minorTickInterval</a> option.</p>\n<h2>Multiple axes</h2>\n<p>It is possible to have multiple axes and linking them with different data series. And the <code>opposite: true</code> option puts the axis on the right side of the chart.</p>\n<h3>Align ticks</h3>\n<p>When using multiple axes, it is usually desirable to align the ticks, to avoid multiple sets of grid lines to mess up the chart. The <a href=\"https://api.highcharts.com/highcharts/chart.alignTicks\" target=\"_blank\">chart.alignTicks</a> option is <code>true</code> by default. The downside of alignment is that each axis is pre-determined to have the same tick amount as the others, so the fitting of ticks may not be optimal. An alternative is to turn off <code>alignTicks</code> and set <code>gridLineWidth</code> to 0.</p>\n<h3>Align thresholds</h3>\n<p>Since v10, it is possible to align the <a href=\"https://api.highcharts.com/highcharts/series.line.threshold\" target=\"_blank\">thresholds</a> of multiple axes through the <a href=\"https://api.highcharts.com/highcharts/chart.alignThresholds\" target=\"_blank\">chart.alignThresholds</a> option. As a consequence, Highcharts will remove 0 or negative points associated to the axis, and if you try to set the <a href=\"https://api.highcharts.com/highcharts/yAxis.min\" target=\"_blank\">axis.min</a> option to 0 or negative, it will fail with an error.</p>\n<h3>Datetime</h3>\n<p>A datetime axis prints labels of round date values in appropriate intervals. However, it is important to understand when to use categories and when you are better off with a linear or datetime xAxis.</p>\n<p>Categories are groups of items, like for example \"Apples\", \"Pears\" and \"Oranges\", or \"Red\", \"Green\", \"Blue\", \"Yellow\". These categories have that in common that there are no intermediate values. There's no sliding transition between apples and pears. Also, if you leave one category out, the user isn't able to understand what is left out. Say if you print every second color of \"Red\", \"Green\", \"Blue\" or \"Yellow\", the user won't know what colors are missing. Therefore, Highcharts doesn't have automatic methods to hide categories if they become to dense on the axis.
