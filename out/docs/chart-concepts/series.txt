series</h2>\n<p>The actual data is represented as an array, by the data attribute, and can be presented in three ways:</p>\n<ol>\n<li>A list of numerical values. In this case, the numerical values will be interpreted as <code>y</code> values and the <code>x</code> values will be automatically calculated, either starting at 0 and incrementing by 1, or from the <a href=\"https://api.highcharts.com/highcharts/series.line.pointStart\" target=\"_blank\">pointStart</a> and <a href=\"https://api.highcharts.com/highcharts/series.line.pointInterval\" target=\"_blank\">pointInterval</a> options. In this case, the first value is the x value and the second is the y value. If the first value is a string, it is applied as the name of the point, and the x value is incremented following the above rules. Some series, <a href=\"https://api.highcharts.com/highcharts/series.arearange.data\" target=\"_blank\">like arearange</a>, accept more than two values. See API documentation for each series type. Example:</li>\n</ol>\n<pre><code>data: [[5, 2], [6, 3], [8, 2]]\n</code></pre>\n<p><a href=\"https://jsfiddle.net/gh/get/library/pure/highcharts/highcharts/tree/master/samples/highcharts/series/data-array-of-arrays/\" target=\"_blank\">Online example</a></p>\n<ol start=\"3\">\n<li>A list of objects with named values.
