title=\"Highcharts Maps Map Collection\" target=\"_blank\">Highcharts Maps Map Collection</a>. For information on drawing your own maps and creating maps from SVG, see our article on <a href=\"https://www.highcharts.com/docs/maps/create-custom-maps-for-highmaps\" title=\"Custom maps\" target=\"_blank\">creating custom maps</a>.</p>\n<p>We see that many users have map data in ESRI Shapefile format or other common mapping formats. These can easily be converted for use with Highcharts Maps using an editor with GeoJSON exporting capabilities, as Highcharts Maps supports the GeoJSON format natively. Most full-featured GIS editors will be able to perform this conversion. <a href=\"https://qgis.org\" title=\"QGIS\" target=\"_blank\">QGIS</a> is a free alternative that supports both Shapefile, KML, and a number of other formats. See <a href=\"https://highcharts.com/maps/demo/geojson\" title=\"GeoJSON demo\" target=\"_blank\">this demo</a> for an example of how to load polygonal data from a GeoJSON file, and <a href=\"https://highcharts.com/maps/demo/mapline-mappoint\" title=\"GeoJSON multiple types demo\" target=\"_blank\">this demo</a> for a more complex example with line and point data.</p>\n<p>The Highcharts Maps map collection is supplied in SVG, GeoJSON and TopoJSON formats, and can be edited using any compatible editor. Simple changes can be made using a text editor. <a href=\"https://inkscape.org\" title=\"InkScape\" target=\"_blank\">InkScape</a> is a free visual editor for SVG files. The map collection is also supplied as executable JavaScript files. These JavaScript files are identical to the GeoJSON files, except for a small amount of wrapping code.</p>\n<p>Please note that different coordinate systems are used in the Map Collection's TopoJSON, GeoJSON and SVG files, mainly for optimization reasons.</p>\n<ul>\n<li><strong>TopoJSON</strong> files, available since Map Collection v1.1.4, are not projected, meaning coordinates are given as <code>[longitude, latitude]</code>. This relies on projection being done in Highcharts, which is more intuitive and allows easier navigation, re-projection, addition of points and lines etc.</li>\n<li><strong>GeoJSON</strong> files in the Map Collection date back to when projection was not supported natively in Highcharts. The coordinates are pre-projected with the projection given in the <code>crs</code> property, in addition to some compression defined in <code>hc-transform</code>. In order to deal with longitude and latitude on the client side, the <code>proj4.js</code> library must be used. Since v9.3, Highcharts has built-in support for projection, so custom GeoJSON maps may dropp pre-projection.</li>\n<li><strong>SVG</strong> files don't deal with geographic information at all. In general, these are only recommended for simple choropleth maps.</li>\n</ul>\n<h2>Creating maps for Highcharts Maps</h2>\n<p>We will now go through the process of importing a Shapefile in QGIS and converting it for use with Highcharts Maps. To do this, select the \"Add Vector Layer\" option from the Layer menu.
