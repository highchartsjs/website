supports solid and semi-transparent colors as well as linear and radial gradients. Unless specified below, this applies to all color options in the Highcharts API, whether it is called color, backgroundColor, borderColor, lineColor etc.</p>\n<h2>Solid colors</h2>\n<p>Primarily, Highcharts supports solid colors given in hex format <em>#00FF00</em> and rgb format <em>rgb(0,255,0)</em>.</p>\n<p>Secondary, any color format that is recognized by the browser, like short Hex <em>#0F0</em> or color names (<em>red, brown, blue</em>) is supported. However, in some cases Highcharts alters the brightness of the color, like when hovering a column chart. When using the secondary color formats, this operation isn't performed. By plugging in to the Highcharts.Color object, we can make named colors work with external libraries or color definitions, like <a href=\"https://jsfiddle.net/gh/get/library/pure/highcharts/highcharts/tree/master/samples/highcharts/chart/colors-parsers/\" target=\"_blank\">this example with RGBColor</a>.</p>\n<h3>Semi-transparent colors - opacity</h3>\n<p>Semi-transparent colors in Highcharts are given in the rgba format <em>rgba(255,255,255,1)</em>. The last parameter is the alpha or opacity that ranges from 0, fully transparent, to 1, fully opaque. Because of this, there are no separate opacity options in the Highcharts API.</p>\n<h2>Linear gradients</h2>\n<p>Linear gradients in Highcharts have a similar syntax to that of SVG:</p>\n<pre><code>color: {\n    linearGradient: { x1: 0, x2: 0, y1: 0, y2: 1 },\n    stops: [\n        [0, '#003399'],\n        [1, '#3366AA']\n    ]\n}\n</code></pre>\n<p>The color is given as an object literal containing two properties:</p>\n<ul>\n<li><strong>linearGradient</strong> holds another object literal that defines the start position (x1, y1) and the end position (x2, y2) relative to the shape, where 0 is top/left and 1 is bottom/right.</li>\n<li><strong>stops</strong> is an array of tuples. The first item in each tuple is the position in the gradient, where 0 is the start of the gradient and 1 is the end of the gradient. Multiple stops can be applied. The second item is the color for each stop. This color can also be given in the rgba format, though in Internet Explorer 8 and less only the first and last opacity will be applied, and intermediate stops will have opacities interpolated between those.</li>\n</ul>\n<p>In modern SVG enabled browsers the linear gradients can be applied to both fills (backgrounds) and strokes (lines). Internet Explorer 8 and less doesn't support gradients on lines, so in these browsers only the first stop color is used.</p>\n<p>Note that linear gradients can be differently defined (as an array or an object). Also, start/end positions might be calculated differently depending on the <code>gradientUnits</code> property (this property can only be set in linear gradient declared as object).</p>\n<p><code>gradientUnits</code> values:</p>\n<ul>\n<li><strong><code>userSpaceOnUse</code></strong> Default when gradient declared as an array. Start and end positions have to be declared as pixels on the chart.</li>\n<li><strong><code>objectBoundingBox</code></strong> Default when gradient declared as an object. Start and end positions are in the range of 0 to 1 as described above. Using this might sometimes result in the disappearance of the coloured element.</li>\n</ul>\n<p>See the online example of a <a href=\"https://jsfiddle.net/highcharts/4rTBY/\" target=\"_blank\">linear gradient chart background</a>.</p>\n<h2>Radial gradients</h2>\n<p>Since Highcharts 2.3 radial gradients have been supported. They have a similar syntax to that of SVG:</p>\n<pre><code>color: {\n    radialGradient: { cx: 0.5, cy: 0.5, r: 0.5 },\n    stops: [\n       [0, '#003399'],\n       [1, '#3366AA']\n    ]\n}\n</code></pre>\n<p>The color is given as an object literal containing two properties:</p>\n<ul>\n<li><strong>radialGradient</strong> holds another object literal, with three properies. The <strong>cx</strong> and <strong>cy</strong> properties are the horizontal and vertical centers of the radial gradient respectively, relative to the shape where 0 and 1 are the edges and 0.5 is the center of the shape. When all values are 0.5, the gradient starts in the middle of the circle and ends along the perimeter.</li>\n<li><strong>stops</strong> is an array of tuples. The first item in each tuple is the position in the gradient, where 0 is the start of the gradient and 1 is the end of the gradient. Multiple stops can be applied.
